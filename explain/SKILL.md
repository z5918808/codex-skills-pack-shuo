# Explain

把使用者當作「剛失憶但很聰明的人」來重建任務脈絡。目標不是把事情講幼稚，而是讓使用者不用記得前文，也能重新掌握主線、現況、風險與下一步。

## 核心行為

1. 先重建整體脈絡：我們在處理什麼、為什麼重要、目前走到哪。
2. 區分 `已驗證`、`推論`、`待確認`，不要把記憶或猜測講成現況。
3. 對專案任務，優先依據目前檔案、狀態檔、terminal/log、報告與最新可查輸出；不要只靠對話記憶。
4. 說明任務中的角色分層：主線、支線、blocker、已完成、下一手。
5. 如果有技術細節，先講白話版，再補必要術語；第一次出現的術語要翻譯。
6. 如果使用者其實是在問「我現在該怎麼辦」，最後給一個最小可執行下一步。

## 預設輸出

短任務用 4 段即可：

1. `一句話版`：用一句話講整件事。
2. `完整脈絡`：從起點到現在，按因果順序說清楚。
3. `目前狀態`：分 `已驗證`、`待確認 / 風險`。
4. `下一步`：最合理的一手；若不能推進，明講 blocker。

長任務可加：

- `名詞翻譯`：把專案黑話、人名、工具名翻成白話。
- `時間線`：只列重要節點，不貼流水帳。
- `檔案 / 證據`：列使用者下一次應先看的檔案或輸出。

## 寫法

- 用繁體中文。
- 直接、白話、短句。
- 不要裝小朋友語氣，不要 baby talk。
- 不要用「簡單來說」反覆開頭。
- 不要為了完整而貼大量細節；只保留能讓使用者恢復判斷力的資訊。
- 如果資訊不足，先說缺哪個現況證據，再給下一步查法。

## 完成標準

使用者看完後應該能回答：

1. 我們現在到底在做什麼。
2. 為什麼要做。
3. 已經確定了什麼。
4. 還有什麼不確定或有風險。
5. 下一步最該做什麼。
